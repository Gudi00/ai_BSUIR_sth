# Полное Техническое Задание (ТЗ) на разработку MVP AI-ассистента для сравнения нормативных актов (Система «НАТАША»)

1. Описание продукта и контекст
Цель: Разработать MVP AI-ассистента для автоматизированного сравнения редакций НПА и ЛНА.
Ключевая ценность: Умное выявление структурных и семантических различий, проверка соответствия иерархии законодательства, полностью объяснимые результаты.

2. Архитектура системы
- Frontend: Next.JS + TailwindCSS + TypeScript.
- Backend: Python (FastAPI). AST-парсинг, Tree Diffing.
- AI Layer: Модель rubert-tiny2 для эмбеддингов, векторный поиск (RAG) с фильтрацией по hierarchy_level.

3. Пользовательский интерфейс (UI/UX)
- Цветовая палитра: Фон #1A1A1A, Акцент #E30613 (MTS Red), Карточки #262626.
- Модуль «База эталонов»: CRUDL управление 10 уровнями иерархии НПА. Поддержка Drag-and-Drop и включения/выключения файлов.
- Дорожка изменений: GitHub-подобный diff. Слева — зачеркнутый старый текст, справа — полужирный новый.
- Юридический вердикт: Блок 🛡 (Shield) при конфликтах с вышестоящими актами.

4. Алгоритм компрессии («Красный хвост»)
Автоматическое свертывание (Collapse) длинных диапазонов (3+ блоков) типа `added`, если в них нет значимых изменений (modified/moved) или триггеров риска.

5. Оценка критичности (Semantic Criticality Score)
Вывод процента (0-100%) необходимости переработки внутренних программ на основе Cosine Similarity и Rule-based детекторов (модальность, сроки, суммы).

6. Принцип объяснимости (4 слоя)
Для каждого изменения выводится:
1. Причина сопоставления (Alignment reason).
2. Описание смыслового изменения (Change summary).
3. Ключевые фрагменты (Highlighting).
4. Итоговый комментарий (Human-like comment).

7. Обработка данных
- AST и Tree Diffing для защиты от сдвига нумерации.
- Semantic Chunking: текст узла + path_context + hierarchy_level.

8. Структура API
Ожидается массив объектов `single_block` и `collapse_group` согласно спецификации в ТЗ.
