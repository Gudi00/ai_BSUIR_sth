[PROJECT CONTEXT]
Проект: "LegalDocComparer" — хакатонный MVP для интеллектуального сравнения двух редакций нормативных или локальных нормативных актов (.docx, .pdf).
Суть: Загрузка двух версий документа, извлечение структуры (разделы, пункты, абзацы), сопоставление блоков (с учетом изменения нумерации, слияний и разделений) и выявление рисков изменения юридического смысла (цветовая маркировка: green, yellow, red).

[ARCHITECTURE & CORE LOGIC DETAILS]
Пайплайн обработки: document parsing -> block normalization -> block alignment -> difference detection -> legal-risk classification -> report generation.
Главные логические правила:

Alignment (сопоставление): Не опираться только на номера! Использовать score, учитывающий: номер, заголовок, лексику (lexical similarity), эмбеддинги, сущности, позицию. Типы связей: 1_to_1, 1_to_many, many_to_1, deleted, added, uncertain.

Классификация рисков (Risk Triggers):

Red (высокий риск): Изменение сроков, чисел/сумм/процентов, модальности (вправе <-> обязан), запретов/разрешений, появление/исчезновение "не", смена субъекта.

Yellow (средний риск/внимание): Значимое текстовое различие без явного триггера, слабая уверенность (uncertain alignment/score).

Green (низкий риск): Редакционные правки, совпадения после лемматизации.
